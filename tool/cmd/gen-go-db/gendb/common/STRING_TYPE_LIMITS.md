# DB2 字符串类型长度范围

以下是 DB2 数据库中各种字符串类型的长度限制：

## 基本字符类型

| 类型 | 长度限制 | 说明 |
|------|----------|------|
| `CHAR(n)` | 最大 254 字节 | 定长字符型，存储 n 个字符，默认右填充空格 |
| `VARCHAR(n)` | 最大 32,672 字节 | 变长字符型，最多存储 n 个字符，动态存储 |
| `LONG VARCHAR` | 最大 32,700 字节 | 变长字符串，比 VARCHAR 更长 |

## 图形字符类型（用于双字节字符）

| 类型 | 长度限制 | 说明 |
|------|----------|------|
| `GRAPHIC(n)` | 最大 127 双字节字符 (254 字节) | 定长图形字符型，用于存储双字节字符（如中文、日文等） |
| `VARGRAPHIC(n)` | 最大 16,336 双字节字符 (32,672 字节) | 变长图形字符型，用于存储双字节字符 |
| `LONG VARGRAPHIC` | 最大 16,350 双字节字符 (32,700 字节) | 变长图形字符型，长度比 VARGRAPHIC 更长 |

## 大对象类型（CLOB）

| 类型 | 长度限制 | 说明 |
|------|----------|------|
| `CLOB` | 最大 2,147,483,647 字节 (约 2GB) | 字符大对象，用于存储大文本数据 |
| `DBCLOB` | 最大 1,073,741,823 字节 (约 1GB) | 双字节字符大对象，用于存储大型双字节字符数据 |

## Go 类型映射

在我们的代码生成工具中，所有这些类型都映射到 Go 的 `string` 类型，因为：

1. Go 的 `string` 类型可以处理任意长度的字符串
2. Go 的字符串是 UTF-8 编码，可以正确处理各种字符集
3. 从数据库查询时，长度限制由数据库层控制，Go 程序只需要接收返回的数据

## 注意事项

- `CHAR` 类型在存储时会用空格填充到指定长度，查询时也会返回填充的空格
- `VARCHAR`、`LONG VARCHAR` 等变长类型只占用实际数据所需的存储空间
- `GRAPHIC` 和 `VARGRAPHIC` 类型专门用于处理双字节字符（如中文、日文、韩文等）
- `CLOB` 和 `DBCLOB` 用于存储大型文本数据，适合存储文档、大段文本等
